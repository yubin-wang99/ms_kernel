# Offline throughput (RPS) вЂ” per-scope batch sweep

Re-label of the per-scope latency sweeps as **offline throughput at batch B** (requests/s). Derived from the per-scope `torch.cuda.Event` latencies (`tests/harness_perscope_results_260625*.jsonl`); **no re-measurement** (RPS = `BВ·1000/total_ms` is an exact transform of B and latency). `mq`=MSAQ, `mx`=MXINT8, `bf`=bf16. **S3 KV uses u4/gs16** (vpack, byte-roofline KV-read).

## Definitions & caveats

- **RPS (req/s)** = `BВ·1000 / total_ms`, `total_ms = ttft + ОЈ decode-step latency`. Each batch retires B requests.
- **decode (tok/s)** = `BВ·L_outВ·1000 / decode_ms` вЂ” pure generation throughput; isolates decode from prefill.
- **Offline/saturated, static batch** вЂ” throughput ceiling at fixed B, not SLA-constrained online RPS.
- **ratio > 1 = MSAQ faster.** For KV scopes (S3вЂ“S6) the win GROWS with L_out: prefill is a format-tie, so the longer the decode, the less the KV win is diluted (compare the two workloads below).
- **Bв‰¤32**: Bв‰Ґ64 OOMs on 24 GB.
